Cheat Sheets

Concise, annotated reference cards for the commands, syntax, and concepts you'll use every day. Each entry includes a short description so you understand not just what to type — but why.

Git Commands
Version Control
git initInitializes a new Git repository in the current folder. Run this once when starting a new project.
git clone <url>Downloads a remote repository to your machine. Use this to get a copy of a project from GitHub.
git add .Stages all changed files for the next commit. Use git add filename to stage one specific file.
git commit -m "msg"Saves a snapshot of your staged changes. Write clear messages like "Add login feature" — future you will thank you.
git push origin mainUploads your local commits to the remote repository so others can see your changes.
git pullDownloads and merges the latest changes from remote. Always pull before you push to avoid conflicts.
git checkout -b <name>Creates a new branch and switches to it. Use branches to work on features without touching the main codebase.
git statusShows which files are changed, staged, or untracked. Run this often — it's your map of where you are.
git log --onelineShows a compact list of all commits. Useful for reviewing history or finding a commit to revert to.
git stashTemporarily saves your uncommitted changes so you can switch branches without losing work. Use git stash pop to restore.

Linux Terminal
Command Line
ls -laLists all files including hidden ones, with permissions and sizes. Hidden files start with a dot.
cd <folder>Changes the current directory. Use cd .. to go up one level, cd ~ to go home.
mkdir -p <path>Creates a directory. The -p flag creates all parent directories if they don't exist yet.
cp -r src/ dst/Copies a folder and its entire contents. Without -r it only works on single files.
rm -rf <folder>Permanently deletes a folder and everything inside. No recycle bin — be very careful with this command.
chmod 755 <file>Sets file permissions. 755 = owner can read/write/execute, everyone else can only read and execute.
grep -r "text" .Searches all files in the current directory for a pattern. Great for finding where a function or variable is used.
kill -9 <PID>Force-stops a process by its ID. Get the PID first using ps aux | grep processname.
SQL Queries
Database
SELECT * FROM tRetrieves all columns from a table. Replace * with specific column names to fetch only what you need.
WHERE col = valFilters rows by a condition. Combine conditions with AND / OR. Always comes after FROM.
INNER JOIN t2 ON keyCombines rows from two tables where keys match. Use LEFT JOIN to keep all rows from the left table even without a match.
GROUP BY colGroups rows with the same value so you can use aggregate functions like COUNT, SUM, or AVG on each group.
ORDER BY col DESCSorts the result set. DESC = largest first, ASC = smallest first. Default is ascending.
INSERT INTO t VALUESAdds a new row. Always specify column names explicitly to avoid errors when the table schema changes.
Python Built-ins
Language
v3.12
len(x)Returns the number of items in a list, string, tuple, or dictionary.
range(start, end)Generates a sequence of numbers for loops. range(0, 5) gives 0, 1, 2, 3, 4 — end is exclusive.
enumerate(iterable)Loops with an automatic index — returns (index, value) pairs. Cleaner than manually incrementing a counter.
zip(a, b)Pairs up two lists element by element. Stops at the shortest list. Great for iterating two lists together.
sorted(list)Returns a new sorted list without modifying the original. Add reverse=True for descending order.
isinstance(x, type)Checks if a variable is of a specific type. Returns True or False. Safer than using type(x) ==.
JavaScript
Language
ES2024
const / letUse const for values that won't change, let for those that will. Avoid var — it has unpredictable scoping.
arr.map(fn)Creates a new array by running a function on every element. Doesn't change the original array.
arr.filter(fn)Returns a new array with only the elements where the function returns true. Great for searching lists.
async / awaitMakes asynchronous code (like fetch API calls) read like normal synchronous code. Wrap in try/catch for error handling.
obj?.propOptional chaining — safely accesses a property without crashing if obj is null or undefined.
{...obj}Spread operator — copies all properties of an object. Use it to merge objects or clone without sharing a reference.
Sorting Algorithms
Algorithms
Bubble SortRepeatedly swaps adjacent elements. Simple to understand but very slow on large data.
O(n²)
Selection SortFinds the minimum element and places it at the front each pass.
O(n²)
Insertion SortFast on nearly-sorted data. Builds the sorted array one item at a time.
O(n²)/O(n)
Merge SortDivide-and-conquer. Stable and consistent. Uses extra memory.
O(n log n)
Quick SortFast in practice. Picks a pivot and partitions around it. Unstable.
O(n log n)
Regex Patterns
Text Matching
^ and $Anchors — ^ matches the start of a string, $ matches the end. Used to match exact strings.
\d \w \sShorthand classes — \d = digit (0–9), \w = word character (a–z, A–Z, 0–9, _), \s = whitespace.
* + ?Quantifiers — * = 0 or more, + = 1 or more, ? = optional (0 or 1 time).
[a-z0-9]Character class — matches any lowercase letter or digit. Use [^abc] to match anything except those characters.
{n,m}Repeat n to m times. {3,16} means at least 3 and at most 16 characters — useful for password validation.
Markdown
Formatting
# HeadingCreates headings. One # = H1 (largest), up to ###### for H6. Used in README files and docs.
**bold** _italic_Double asterisks for bold, single underscore or asterisk for italic. Can be combined: **_bold italic_**.
`code` / ```block```Single backtick for inline code, triple backticks for a fenced code block. Add the language name for syntax highlighting.
[text](url)Creates a hyperlink. The bracketed text is displayed, the URL in parentheses is where it points.
- item / 1. itemCreates bullet lists (use - or *) or numbered lists. Indent 2 spaces to nest items.
